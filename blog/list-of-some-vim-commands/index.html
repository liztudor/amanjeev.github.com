List of some Vim commands

Just a list, nothing special.

=

Reformats the selected text according to the current indent rules.

==

Auto-indent current line.

g-

If you make a lot of edits, then undo them, then make one more edit, then decide you want the first set of edits back, most editors can't do it. Emacs can, but it's a pain. g- in Vim will usually take you straight there. This is not available in Vi.

%

Jump to the corresponding sibling item like open/close brackets - (), {}, []. Or HTML tags.

zz

Center your screen on the current line.

gi

In command mode, this goes to insert mode at the last place you were in the insert mode.

Recording with : q<character>

Replay with : @<character>

Example, record with qa and replay with @a.

:so %

To reload the file you are currently editing. This helps when you are editing .gvimrc or .vimrc files in Vim and you want to reload the changes.

:vsp

:sp

Vertical split and horizontal split window, respectively. A lot of combinations are available. They take a command to not duplicate the window by default and open the one you specify.

set splitbelow

set splitright

To make splitting more intuitive.

nnoremap <C-J> <C-W><C-J>

nnoremap <C-K> <C-W><C-K>

nnoremap <C-L> <C-W><C-L>

nnoremap <C-H> <C-W><C-H>

To map moving between the buffer splits more intuitive with c-hjkl.

ctrl + w _

Max out the height of the current split

ctrl + w |

Max out the width of the current split

ctrl + w =

Normalize all split sizes, which is very handy when resizing terminal

Ctrl+W R

Swap top/bottom or left/right split

Ctrl+W T

Break out current window into a new tabview

Ctrl+W o

Close every window in the current tabview but the current one
