Pelican + Foundation

Hello, World!

This is an absolutely new site which is statically generated using Pelican. The framework used is Foundation.

Pelican requires a theme which can be anything but I created a theme from Foundation - [Github Repository] and here is the final source for amanjeev.com (this site).

The only trouble I had was that Pelican by default requires the home page to be a blog. That means, the home page is paginated. There is a solution that makes it possible to have home page as a static page (reST or Markdown) and move your blog page to some other URL like /blog.html. I could have used it but I saw two opportunities in the code -

  1. To have the home page still be a dynamic page with dynamic listing of the latest posts.
  2. To be able to specify all the new templates that you add as paginated.

So, I made some changes which are clearly not official and live on a separate branch. There is indeed a lot of discussion needed and the Gods of the project will know more than I do at this point.

However, there is some discussion that has started on the archive pages. I clearly think that the archive pages need pagination and the default page limit value must be different than that of the normal pagination for the blog posts. Archives page has considerably less data so a default value of DEFAULT_PAGINATION, which is usually around 5 to 10, does not work for it.

Github issue in case you want to chime in.
